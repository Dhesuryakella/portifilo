Decentralized Swarm Robotics System

NITK Surathkal Research | IEEE Published

Problem

Centralized multi-robot systems fail due to communication latency and single-point failure. Needed a robust, low-latency coordination method for formation control.

My Solution

Designed and implemented a decentralized leader–follower swarm using ESP32-based robots communicating via ESP-NOW, coordinated with a TurtleBot leader running ROS. Built ArUco marker-based vision localization for relative positioning.

My Role

System architecture, ESP32 firmware development, ESP-NOW protocol implementation, ROS integration, ArUco localization, and experimental evaluation.

Tech Stack

ESP32 ESP-NOW ROS (Noetic) ArUco Markers OpenCV C/C++ Python

Outcome

  • ✓ Real-world multi-robot experiments completed
  • ✓ Achieved <100ms communication latency
  • ✓ Latency and spacing analysis validated
  • ✓ Results published in IEEE ICRM 2025 conference paper

ESP32 Multi-Node Communication System

Low-Latency Wireless Firmware

Problem

Standard wireless protocols (Wi-Fi, BLE) had too much latency and overhead for real-time multi-robot coordination requiring sub-10ms response times.

My Solution

Developed custom Embedded C firmware using ESP-NOW protocol for peer-to-peer communication. Implemented UART-based debugging, packet acknowledgment, timing synchronization, and error handling.

My Role

Complete firmware development, protocol design, debugging infrastructure, and performance validation.

Tech Stack

ESP32 Embedded C ESP-NOW UART FreeRTOS

Outcome

  • ✓ Achieved <10ms latency between nodes
  • ✓ Debugged and resolved packet loss issues
  • ✓ System validated through 1000+ transmission tests
  • ✓ Documented performance metrics and limitations

Self-Balancing Two-Wheeled Robot

PID Control with Real Sensor Noise

Problem

Implementing stable PID control is easy in simulation, but real IMU sensors have noise, drift, and calibration issues that theory doesn't cover.

My Solution

Built a self-balancing robot using MPU6050 IMU and ESP32. Implemented sensor calibration, drift compensation using complementary filter, and real-time PID tuning via Bluetooth serial.

My Role

Complete hardware integration, sensor calibration, control algorithm implementation, and iterative tuning.

Tech Stack

ESP32 MPU6050 IMU PID Control Bluetooth Motor Drivers

Outcome

  • ✓ Stable balance achieved on uneven surfaces
  • ✓ Real-time tuning interface for PID parameters
  • ✓ Practical understanding of control theory vs. real hardware
